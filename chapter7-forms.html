Forms - How to collect information from visitors, different kids of form controls and new HTML5 form controls.

Form control live inside the form element (open and close tags). This element should always carry the "Action" attribute and usually has method and id as well.

The action element requires an action attribute. Its value is the URL for the page on the server that will receive the information in the form when it is submitted.

Two types of methods: Get and Post

Get Method: The values from the form are added to the end of URL specified in the action attribute and is ideal fro short forms and when you are just retrieving data from the web server (not when you are adding/deleting info from a database)

Post method are sent in HTTP headers. As a rule of thumb you should use the post method if your form allows users to upload a file, is very long, contains sensitive data, adds information to, or deletes info from a database.

If no method is selected, get is default.

Button and Hidden Controls
The button element was introduced to allow users more control over how their buttons appear, and to allow other elements to apear inside the buttone. This means that you can combine text and images between the opening and closing tags. Effectively a DIY designed button combining text and an image into one button.

Labeling Form Controls

When introducing form controls the code was kept simple by indicating the purpose of each on in text next to it. However, each form control should have its own "label" element as this makes the form accessible to vision impaired users.
The "label element can be used in two ways:
1. Wrap around both the text description and the form input.
2. Be kept separate from the form control and use the for attribute to indicate which form control it is a label for.

Fieldset groupings

You can group related form controls together inside the "fieldset" element. Helpful for longer forms. Most browsers will show the fieldset with a line around the edge to show how they are related. The appearance of these lines can be adjusted using CSS.
The "legend" element come directly after the opening "fieldset" tag and contains a caption which helps identify the purpose of that group of form controls

Form Validation

Validation typically has been done using js but HTML5 allows it to be done in HTML in the browser.
Validating helps ensure the info entered will be understood by he server receiving the form which reduces the work the server has to do and enables users to see if there are any problems with the form faster than if the validation was done server side.


Date Input
